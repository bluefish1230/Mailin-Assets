# 公司資產清單 - 雲端管理系統 (Mailin Assets)

本專案為公司開發的「回應式財產清單管理系統」，支援桌機、平板、手機跨平台檢視。系統整合了 Firebase 雲端資料庫，具備資產編號自動跳號、雲端報廢管理、管理員權限登入、以及手機手寫確認簽收功能。

## ✨ 核心特色
*   **雲端即時同步**：採用 Firebase Firestore，資產的新增、編輯、刪除與報廢皆即時寫入雲端。
*   **管理員權限保護**：進入系統須通過身分驗證，登入後方可執行管理操作。
*   **資產分類編碼 (連續不重複)**：
    *   `PC`：電腦 (PC001...)
    *   `NB`：筆電 (NB001...)
    *   `N`：其它類別 (N001...)
    *   系統會自動計算目前最大流水號（包含已報廢資產），確保編號始終順延。
*   **手寫簽名連結**：管理員可按類別篩選並批次產生專屬手機簽名連結，使用者開案即可隨時手寫確認簽收。
*   **資產異動紀錄**：自動追蹤保管人的變更歷程，支援查詢歷史變更的時間、原保管人與新保管人。
*   **報廢清冊管理**：詳細紀錄資產報廢時的規格、採購日期與報廢時間。
*   **系統重置工具**：儀表板內建開發者工具，可一鍵清空所有測試紀錄（需二次確認）。

## 🚀 部署指引

1. **前端預覽與發佈**
   - 本專案使用 Vanilla HTML/JS/CSS，無需編譯。
   - 檔案上傳至 GitHub 儲存庫後，於 Settings > Pages 開啟發佈。

2. **資料庫結構 (Firebase Firestore)**
   - `assets`：在用資產 (包含編號、品名、規格、保管人、地點、採購日)。
   - `scrapping`：報廢紀錄。
   - `signatures`：手寫簽名圖片及關聯資產 ID。
   - `transfers`：保管人異動日誌。

## 📋 系統規格與操作
*   **主要欄位**：
    *   資產編號 (自動生成)
    *   品名
    *   規格 (例如: CPU/RAM 配置)
    *   保管人
    *   存放地點
    *   採購日期 (Date Input)
*   **批次操作**：支援「批次修改地點/保管人」與「批次報廢」。
